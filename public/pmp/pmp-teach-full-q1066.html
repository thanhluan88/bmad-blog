A healthcare company is sponsoring a project to develop a new mobile application. The project must comply with strict industry regulations and meet a fixed launch deadline mandated by regulators. The project manager knows that governance thresholds are essential to track performance and risk exposure. If thresholds are not realistic, missed milestones or compliance failures could result in regulatory fines and reputational harm to the company. What should the project manager do?

Solution gốc (CSV — cột P)

B. Define key performance indicators (KPIs) and establish acceptable limits that meet regulatory compliance. The project manager should use KPIs plus explicit limits to set performance and risk thresholds that result in regulatory compliance. The other options are incorrect. High-level appetite/tolerance is useful context, but not for project-specific measurable thresholds. EVM is a monitoring tool. It does not define thresholds and reacts after the issue occurs. Review helps, but without measurable linkage, it fails to create actionable thresholds.
