Practice Questions — PMBOK 8th Ed

Bài học Q73 về Stakeholder Register — Caleb tại HL Infrastructure, coastal bridge repair: giữa kickoff, fishermen's association phàn nàn không được consult, delay permits. Câu hỏi hỏi Caleb nên làm gì first.

PMBOK Guide 8, p.71 & 141–142 Identify Stakeholders Stakeholder Register Câu 73 — Practice

1. Stakeholder register (p.141–142)

Project document chứa stakeholder information: identification details, assessment of expectations and influence, classification. Ghi roles, contact details, impact, categories (internal/external) — hỗ trợ analysis và engagement suốt project.

Instructor: New stakeholder → register first

Khi stakeholder mới xuất hiện (complaint, delay permits), bước đầu tiên là add/update stakeholder register — document họ trước khi plan engagement hoặc comms.

2. Thứ tự hành động — "FIRST"

  1. Add to stakeholder register — document new stakeholder (C)
  2. Conduct stakeholder analysis — assess interests, influence, impact (D)
  3. Develop/update stakeholder engagement plan (A)
  4. Update communications management plan (B)
OptionActionFIRST?
A. Engagement plan Develop strategies to engage No — after identify + register
B. Comms plan Update how/when to communicate No — follows engagement planning
C. Stakeholder register Add fishermen's association Yes — document first
D. Stakeholder analysis Analyze with the association No — register before deeper analysis

Q72 vs Q73

Q72: boundary Identify vs Plan Engagement. Q73: khi stakeholder mới xuất hiện, register là bước đầu tiên — trước analysis session (D) và plans (A, B).

3. Câu hỏi thực hành — Question 73

Practice Questions PMBOK 8th Ed · Q73
Caleb is launching a coastal bridge repair project at HL Infrastructure. Midway through kickoff activities, a local fishermen's association files a complaint after not being consulted, delaying permits. What should Caleb do first?

4. Phân tích đáp án — Đáp án đúng: C

Tại sao chọn C?

  • New stakeholder discovered → update stakeholder register immediately.
  • Register captures ID, expectations, influence, classification (p.141–142).
  • "First" = document before planning engagement or conducting analysis.

Loại trừ từng đáp án

Đáp ánTại sao không FIRST
A. Engagement plan Requires known stakeholders in register — planning step, not first.
B. Comms plan Communication approach follows stakeholder identification and engagement planning.
D. Stakeholder analysis Sounds proactive but PMI wants register updated first — analysis builds on documented stakeholder.

5. Drill — FIRST hay LATER?

New community group complains — add them to stakeholder register
Develop tailored engagement strategy for resistant stakeholder
Update communications management plan for new audience
Facilitate stakeholder analysis workshop with the association

0 / 4 đúng

6. Bẫy thi

Bẫy 1: Complaint → jump to analysis (D)

Analysis with the association sounds like the right fix — but PMI sequence: register first, then analyze and plan.

Bẫy 2: Delay → engagement plan (A)

Engagement plan is needed eventually — not before the stakeholder is documented in the register.

Bẫy 3: Communication issue → comms plan (B)

"Not consulted" suggests comms — but identify/document stakeholder first, then update plans.

Pattern

New stakeholder · complaint · missed consultation → update stakeholder register FIRST (C).

7. Flashcard

Nhấn để lật

Stakeholder register contains?
ID details, expectations, influence, roles, contact, impact, internal/external classification.
Q73 đáp án?
C — Add them to the stakeholder register (FIRST).
