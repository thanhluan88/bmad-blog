Practice Questions — PMBOK 8th Ed

Bài học Q27 về Change Log — nhiều modifications được request, team confused về what to implement and in what sequence trong execution.

PMBOK Guide 8, p.22 & p.116 Manage Project Execution Câu 27 — Practice

1. Change Log

Change log ghi lại tất cả change requests đã submit và trạng thái của chúng — disposition của từng request xuyên suốt project lifecycle.

Định nghĩa (PMBOK p.116)

A change log records all submitted changes and their status, documenting the disposition of change requests throughout the project.

Manage Project Execution (p.22)

Leading and performing work defined in the plan and implementing approved changes to meet objectives — deliverables meet scope requirements.

2. Phân biệt các log/register

ArtifactGhi gì?Q27 fit?
Change Log Submitted changes + status + disposition + sequence of approved mods ✓ Đúng — modifications throughout project
Scope Statement / Baseline Authorized scope ban đầu — không track history từng change ✗ Thiếu status/sequence của modifications
Issue Log Impacts đã xảy ra cần xử lý ✗ Không phải change request tracking
Risk Register Uncertainties & planned responses ✗ Không liên quan modifications confusion

3. Câu hỏi thực hành — Question 27

Practice Questions PMBOK 8th Ed · Q27
Rina is executing a warehouse automation project at Grid Line Logistics. Several modifications have been requested throughout the project, and now there is confusion about what should be implemented and in what sequence. What will Rina do next?

4. Phân tích đáp án — Đáp án đúng: B

Tại sao chọn B?

  • “Modifications requested throughout” = change requests cần track.
  • Confusion về implement gì, thứ tự nào → Change Log có status & disposition.
  • Execution phase — implement approved changes per log.

Loại trừ từng đáp án

Đáp ánTại sao không đúng
A. Scope Statement Baseline scope ban đầu — không document lịch sử nhiều modifications & sequence.
C. Issue Log Issues = đã xảy ra — không phải track change requests.
D. Risk Register Risks = uncertainty — không giải quyết confusion về approved modifications.

Liên hệ Q20 Assumption Log

Q20 phân biệt assumption/risk/issue logs ở planning. Q27 = execution — change log khi nhiều modifications gây confusion.

5. Drill — Log nào?

Track status of submitted change requests
Document problem that has already occurred
Uncertain event and planned response
Confusion about approved mods and sequence

0 / 4 đúng

6. Bẫy thi

Bẫy 1: “What to implement” → Scope Statement (A)

Scope baseline = authorized scope; nhiều modifications cần Change Log để biết approved changes & order.

Bẫy 2: Confusion → Issue Log (C)

Issue = đã materialize — đây là change requests chưa rõ implementation sequence.

Pattern

Modifications + status/sequence confusion → Change Log (B).

7. Flashcard

Nhấn để lật

All submitted changes + status + disposition throughout project.
Q27 đáp án?
B — Review the Change Log.
