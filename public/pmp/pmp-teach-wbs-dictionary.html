1. WBS Dictionary (Develop Scope Structure, p.42)

WBS decomposes total scope into manageable work packages. The WBS dictionary adds detailed documentation for each component: description, assumptions, constraints, responsible org, milestones, resources, cost estimates, acceptance criteria.

WBS Dictionary vs WBS

WBS = hierarchical structure. Dictionary = detailed narrative per work package so team can assign, track, and measure with shared understanding.

Typical dictionary fieldsPurpose
DescriptionWhat the work package includes
Responsible / ResourcesWho does the work
Criteria / CostAcceptance criteria, estimates
Assumptions, constraints, milestonesPlanning clarity

3. Câu hỏi thực hành — Question 52

Practice Questions PMBOK 8th Ed · Q52
Nate is managing a manufacturing facility upgrade using a predictive approach. Nate has elicited the requirements from project stakeholders and turned them into project scope. Nate wants to add more details to the work packages to ensure clarity and alignment across the team. What will he do next?

4. Phân tích đáp án — Đáp án đúng: D

Tại sao chọn D?

  • Work packages already exist (scope → WBS) → add detail via WBS dictionary.
  • Clarity & alignment → description, criteria, responsible party per package.
  • Develop Scope Structure (p.42) — predictive decomposition + dictionary.

Loại trừ từng đáp án

Đáp ánTại sao không đúng
A. RAM RACI/RAM maps roles to deliverables — sau khi work defined, không thay dictionary detail.
B. Quality control Verify completed work during execution — không define package details in planning.
C. RTM Trace requirements to scope (Q44) — không add narrative detail to work packages.

Liên hệ Q51 vs Q52

Q51: organize deliverables into WBS. Q52: more details on work packages → WBS dictionary (D). Đừng nhầm với RTM (requirements) hay RAM (roles only).

5. Drill — WBS, dictionary, RTM hay RAM?

Add details to work packages for clarity
Decompose deliverables into manageable components
Link requirements to scope components
Map who is Responsible, Accountable, Consulted, Informed

6. Bẫy thi

Bẫy 1: Roles → RAM (A)

RAM defines R/A/C/I — stem cần detail ON work packages (description, criteria), không chỉ roles.

Bẫy 2: Work packages → QC verify (B)

Quality control = execution/monitoring — planning step là dictionary, không verify.

Bẫy 3: Requirements → RTM (C)

RTM traces requirements — Q52 đã có scope; cần dictionary cho work packages.

Pattern

More details to work packages · clarity · alignment · predictive → WBS dictionary (D).
