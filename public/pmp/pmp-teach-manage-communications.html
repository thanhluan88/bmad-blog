PMBOK Guide 8, p.75 Manage Communications Push vs Pull · PMIS Câu 79 — Practice

1. Push vs Pull (Manage Communications, p.75)

Manage Communications ensures timely collection, creation, distribution, and management of project information. Tools include PMIS, project reporting, and push/pull communication methods.

Push Sender distributes info to recipients (email, reports sent weekly)
Pull Recipients access info when needed (PMIS repository, portal)

Instructor: Push failed → try Pull

Weekly reports = push already happening — stakeholders still not updated. Solution: PMIS + pull so they access latest info on demand (A).

2. So sánh các đáp án

OptionActionQ79 fit?
A. PMIS + Pull Central repository, stakeholders pull when needed Yes — switch method after push failed
B. PMIS + Push Same push via different channel Still push — weekly send already not working
C. Stakeholder meeting Ensure everyone informed in session One-time — doesn't fix ongoing info access
D. Stakeholder register Update info needs in register Identify artifact — not distribution method

Plan vs Manage Communications

Q76: plan how to communicate. Q79: manage distribution — when push doesn't work, use pull via PMIS.

3. Câu hỏi thực hành — Question 79

Practice Questions PMBOK 8th Ed · Q79
You are managing the Helios Energy rollout project. Several stakeholders complain they are not being updated with the latest project information, despite you sending the project report each week. What will you do next?

4. Phân tích đáp án — Đáp án đúng: A

Tại sao chọn A?

  • Weekly report = push already tried — stakeholders still not updated.
  • Pull via PMIS lets stakeholders access latest info when they need it.
  • Manage Communications tool: PMIS + pull communication method (p.75).

Loại trừ từng đáp án

Đáp ánTại sao không đúng
B. PMIS + Push Still pushing — stem says weekly send isn't reaching/updating them effectively.
C. Meeting Instructor marks X — one-time inform, not sustainable info flow.
D. Register Documents stakeholder info — doesn't solve distribution/access problem.

5. Drill — Push hay Pull?

Stakeholders miss weekly emails — post reports to PMIS portal for self-access
Send status report attachment to all stakeholders every Friday
Team retrieves latest risk log from shared project repository
PM broadcasts urgent change notice to distribution list

0 / 4 đúng

6. Bẫy thi

Bẫy 1: PMIS → assume Push (B)

PMIS works with both — key is pull when push (weekly send) already failed.

Bẫy 2: Not updated → meeting (C)

Meeting informs once — doesn't fix ongoing access to latest project info.

Bẫy 3: Stakeholder → register (D)

Register is identify/plan artifact — stem needs distribution method change.

Pattern

Already sending reports · still not updated → PMIS + pull approach (A).
