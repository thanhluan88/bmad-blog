1. Plan Scope Management

Mục tiêu là tạo scope management plan — định nghĩa cách project sẽ được delivered: establish all work required, eliminate unnecessary work không add value.

Plan Scope Management (PMBOK p.40)

Creates a scope management plan that defines how the project will be delivered by establishing all work required to complete the project and eliminating or removing unnecessary work that will not add value.

3. Câu hỏi thực hành — Question 42

Practice Questions PMBOK 8th Ed · Q42
You are a project manager at SBC, preparing for a commercial building project. Before defining detailed requirements, leadership wants the team to follow a consistent approach to ensure only value-adding work is performed. What should be the next step?

4. Phân tích đáp án — Đáp án đúng: B

Tại sao chọn B?

  • Before requirements → Plan Scope Management first.
  • Consistent approach + value-adding work → scope management plan defines HOW.
  • Eliminate unnecessary work — core objective of Plan Scope Management (p.40).

Loại trừ từng đáp án

Đáp ánTại sao không NEXT
A. Create WBS Sau plan & requirements — WBS decomposes defined scope.
C. Collect requirements Đề nói before detailed requirements — plan trước.
D. Validate scope Monitor/control — acceptance of deliverables, quá muộn.

Liên hệ Q10 Focus on Value & Q28 Scope Baseline

Q10: focus on value. Q42: scope plan ensures only value-adding work. Q28: execute per scope baseline — much later in lifecycle.

5. Drill — Bước nào trong scope sequence?

Before detailed requirements — consistent approach
Gather stakeholder needs after plan exists
Decompose project work into deliverables
Eliminate unnecessary non-value-adding work

6. Bẫy thi

Bẫy 1: Define work → WBS (A)

WBS comes after scope plan & requirements — đề explicitly before detailed requirements.

Bẫy 2: Value → Collect requirements (C)

Requirements collection sau plan — leadership wants approach first.

Bẫy 3: Stakeholders → Validate scope (D)

Validate scope = confirm acceptance — execution/monitoring, not preparing phase.

Pattern

Before requirements + consistent approach + value-adding work → Develop scope management plan (B).
