Thuật ngữ: Alternatives Analysis, Decision Tree Analysis, Kano Analysis

  1. A method used to evaluate identified options in order to select the options or approaches to use to perform the work of the project.
    → A. Alternatives Analysis
  2. A technique used to model and analyze product features by considering the features from the viewpoint of the customer.
    → C. Kano Analysis
  3. A diagramming and calculation technique for evaluating the implications of a chain of multiple options in the presence of uncertainty.
    → B. Decision Tree Analysis

Giải thích dễ hiểu

  • Tình huống: Ghép 3 kỹ thuật phân tích (Alternatives / Kano / Decision Tree) với định nghĩa tương ứng.
  • Đáp án đúng (A, C, B): A. Alternatives Analysis · C. Kano Analysis · B. Decision Tree Analysis (theo thứ tự mô tả 1→3)
  • Vì sao: Alternatives = chọn options/approaches thực hiện work; Kano = feature theo góc nhìn khách hàng; Decision Tree = chuỗi quyết định khi có bất định → thứ tự A,C,B.
  • Mẹo: Alternatives ≠ Decision Tree; Kano luôn gắn customer viewpoint / product features.
  • PMBOK 8: Process: Manage Project Execution · Principle: Lead accountably

Chi tiết tiếng Anh — Why this answer

1. Đáp án tham chiếu (Reference solution)

  • A,C,B. Slot1=Alternatives Analysis; Slot2=Kano Analysis; Slot3=Decision Tree Analysis.
  • Do not swap Alternatives with Decision Tree: one selects approaches, the other models uncertainty chains.

2. Lý luận PMBOK 8 (PMBOK reasoning)

  • Scenario: Match analysis technique definitions used in scope / product decisions.
  • PMBOK 8 Scope domain supports selecting approaches and evaluating product features with stakeholder/customer viewpoint.
  • Therefore A,C,B matches the three definition slots.

Answer

A. Alternatives Analysis · C. Kano Analysis · B. Decision Tree Analysis (theo thứ tự mô tả 1→3)
