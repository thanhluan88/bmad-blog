1. Requirements Traceability Matrix

RTM provides structured way to track requirements from multiple sources — ensuring alignment, visibility, and completeness across the project.

Requirements traceability matrix (PMBOK p.131–132)

Tracks requirements from multiple sources, ensuring alignment, visibility, and completeness. Links requirements to scope, deliverables, and test criteria.

3. Câu hỏi thực hành — Question 44

Practice Questions PMBOK 8th Ed · Q44
Two parallel project streams at NT Industries are being combined into a single initiative. One uses iterative practices, while the other follows a detailed planning approach. After integration, there is confusion about the varied stakeholder requirements and how the team will deliver them as project scope. What should the project manager do?

4. Phân tích đáp án — Đáp án đúng: A

Tại sao chọn A?

  • Multiple sources (iterative stream + predictive stream) → RTM tracks all.
  • Confusion about varied requirements → visibility & alignment via RTM.
  • Deliver as project scope → trace requirements to scope/deliverables.

Loại trừ từng đáp án

Đáp ánTại sao không đúng
B. Single product backlog Agile artifact — không address predictive stream + scope traceability across both.
C. New WBS Decomposes scope — chưa align/track varied requirements from multiple sources.
D. Requirements mgmt plan Plan how to manage — không resolve confusion về existing varied requirements.

Liên hệ Q43 vs Q44

Q43: elicit/gather (focus groups, benchmarking). Q44: track & align multiple sources (RTM) — đừng nhầm RTM với elicitation techniques.

5. Drill — Elicit, trace hay decompose?

Multiple sources — align varied requirements to scope
Differing expectations — collect stakeholder needs
Decompose approved scope into work packages
Ensure completeness & visibility across streams

0 / 4 đúng

6. Bẫy thi

Bẫy 1: Iterative stream → Product backlog (B)

Backlog là agile tool — hybrid integration cần RTM trace multiple sources to scope.

Bẫy 2: Project scope → WBS (C)

WBS sau requirements aligned — confusion cần traceability trước decompose.

Bẫy 3: Requirements → Update plan (D)

Requirements management plan = HOW to manage — không track existing varied reqs.

Pattern

Multiple sources + varied requirements + deliver as scope → Requirements traceability matrix (A).
