Practice Questions — PMBOK 8th Ed

Bài học Q58 về Resource Leveling — FI Group: critical activities cần cùng một specialized engineer overlapping times → unrealistic workload, risk delays. PM nên làm gì?

PMBOK Guide 8, p.50 & 193 Develop Schedule Resource Leveling Câu 58 — Practice

1. Resource leveling (p.50, 193)

Resource leveling adjusts the project schedule to address resource constraints — đảm bảo resources không bị over-allocated và work được scheduled theo actual availability. Có thể reschedule critical activities (có thể extend project end date).

Khi nào dùng leveling?

Same resource assigned to multiple activities at the same time — especially on critical path — → level schedule để eliminate unrealistic workload.

2. Leveling vs Smoothing vs Compression

Resource leveling (A) ✓

Fix over-allocation; may change critical path/end date. Overlapping same resource → level.

Resource smoothing (D)

Adjust within float only; does not change project end date. Không fix critical over-allocation.

TechniquePurposeQ58 fit?
Resource leveling (A) Resolve over-allocation / constraints Yes — overlapping engineer on critical work
Fast tracking (B) Overlap phases to shorten schedule Worsens resource conflict — opposite need
Risk brainstorm (C) Identify & plan responses PMI prefers direct technique when known solution exists
Resource smoothing (D) Balance demand using float Insufficient when critical activities overlap same resource

3. Câu hỏi thực hành — Question 58

Practice Questions PMBOK 8th Ed · Q58
A project schedule at FI Group shows several critical activities requiring the same specialized engineer at overlapping times. This is creating unrealistic workload expectations and risking delays in the future. What should the project manager do?

4. Phân tích đáp án — Đáp án đúng: A

Tại sao chọn A?

  • Over-allocation of one specialized resource across critical activities.
  • Resource leveling reschedules work per actual availability (p.193).
  • Develop Schedule technique — direct fix, không chỉ document as risk.

Loại trừ từng đáp án

Đáp ánTại sao không đúng
B. Fast tracking Overlap activities to compress — tăng resource conflict, không giải quyết over-allocation.
C. Risk brainstorm Problem đã rõ (resource constraint) — apply leveling, không chỉ raise risk.
D. Resource smoothing Chỉ shift within float; critical overlapping work cần leveling, không đủ smoothing.

Instructor diagram

Overlapping tasks (stacked) → sequential reschedule = leveling. Smoothing = move within float without changing end date — different problem.

6. Bẫy thi

Bẫy 1: Delays → Fast tracking (B)

Fast-track compresses schedule — stem là resource over-allocation, không phải cần shorten.

Bẫy 2: Risk of delays → Brainstorm (C)

PMI: known technique (leveling) beats generic risk meeting khi constraint rõ ràng.

Bẫy 3: Resource → Smoothing (D)

Smoothing không đổi end date — critical overlap cần leveling, có thể extend schedule.

Pattern

Same resource · overlapping · over-allocated · critical activities → Resource leveling (A).

7. Flashcard

Nhấn để lật

Leveling vs Smoothing?
Leveling fixes over-allocation (may extend). Smoothing uses float only (same end date).
Q58 đáp án?
A — Apply resource leveling.
