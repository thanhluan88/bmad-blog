PMBOK Guide 8, p.118 Delivery Configuration Management Plan Câu 103 — Practice

1. Configuration Management Plan (p.118)

Configuration management plan (p.118) định nghĩa procedures để track project artifacts và control changes (ví dụ: version control). Plan mô tả items under configuration control, cách identify, record, update, và report changes — đảm bảo product/service/result consistent and functional.

Develop configuration management plan (A — đúng) Multiple document versions · inconsistencies · rework · cần version control và track artifacts → configuration management plan.
Change management plan (B — bẫy) Quy trình submit/approve/reject changes (Q38) — không định nghĩa version control procedures cho design documents/artifacts.
Scope management plan (C — bẫy) Cách define, validate, control scope — không phải document versioning và configuration of artifacts.
Change log (D — bẫy) Ghi nhận changes đã xảy ra (Q27) — output/record, không thiết lập hệ thống version control và configuration procedures.

Signal trong stem Q103

Synapse Biotech · medical device · multiple versions of design documents · inconsistencies · rework · changes not tracked · establish control over versions · proper handling of updates → Configuration management plan (A) — version control + artifact tracking. Không change mgmt plan alone (B), không scope plan (C), không change log (D).

2. So sánh: Q38 Change Plan vs Q103 Configuration Plan (Changes vs Versions)

CâuArtifactMục đíchSignal trong stem
Q38 — Change Mgmt Plan (B) Change management plan How to submit, review, approve changes Stakeholder resistance · formal change process
Q103 — Config Plan (A) Configuration management plan Version control · track artifacts · configuration items Multiple versions · inconsistencies · version control
Q27 — Change Log (D) Change log Record changes that occurred Document what changed · audit trail

Version control vs Change control

Configuration management plan (A): Which version of design documents is current? Track artifacts, identify configuration items, version control — Q103 signal.

Change management plan (B): How to process change requests through ICC — approve/reject baseline changes (Q38). Complements config plan nhưng không thay thế version control.

Change log (D): Ghi lại changes — sau khi đã có process. Không establish version control.

Scope plan (C): What is in/out of scope — không document versioning.

3. Câu hỏi thực hành — Question 103

Practice Questions PMBOK 8th Ed · Q103
Paige is managing a medical device project at Synapse Biotech. Multiple versions of design documents are being used by different teams, causing inconsistencies and rework. Changes are not clearly tracked or controlled. Paige wants to establish control over versions and ensure proper handling of updates. Which approach should she use?

4. Phân tích đáp án — Đáp án đúng: A

Tại sao chọn A?

  • Configuration management plan (p.118) định nghĩa procedures track artifacts và control changes — version control.
  • Stem: multiple versions, inconsistencies, rework → cần single source of truth cho design documents qua configuration control.
  • Plan describes items under configuration control, how identified/recorded/updated, how changes reported.
  • Medical device context — artifact consistency critical; configuration management ensures product remains consistent and functional.

Loại trừ từng đáp án

Đáp ánTại sao không đúng
B. Change management plan Q38 — quy trình manage change requests. Không định nghĩa version control cho design documents và configuration items.
C. Scope management plan Scope definition/control — không address multiple document versions và artifact tracking.
D. Change log Q27 — record changes. Không establish version control system — chỉ log sau khi change xảy ra.

Q27 Change Log · Q38 Change Plan · Q103 Configuration Plan

Q27: Document changes → change log.
Q38: Formal change process → change management plan.
Q103: Version control · artifact tracking → configuration management plan.

5. Drill — Configuration, Change Plan, Scope, hay Change Log?

Synapse: multiple design doc versions, inconsistencies, need version control (Q103)
PM defines which artifacts are under configuration control and version numbering rules
Stakeholders resist changes — PM needs formal submit/approve process (Q38)
PM records each approved change with date, description, and status (Q27)
PM updates how scope will be defined, validated, and controlled on the project

0 / 5 đúng

6. Bẫy thi

Bẫy 1: "Changes not tracked" → Change log (D)

Change log ghi changes — không establish version control. Multiple versions + inconsistencies → configuration management plan (A).

Bẫy 2: "Changes" → Change management plan (B)

Change plan = ICC process. Version control của artifacts = configuration management, không phải change plan alone.

Bẫy 3: Documents → Scope plan (C)

Scope plan không quản lý document versions. Configuration plan track artifacts và version control.

Bẫy 4: Nhầm Q38 và Q103

Q38: change resistance → change management plan. Q103: multiple versions · rework → configuration management plan.

Pattern

Multiple versions · design documents · inconsistencies · rework · version control · artifacts tracked · configuration items → Develop configuration management plan (A).
