Practice Questions — PMBOK 8th Ed

Bài học Q16 về Hybrid Approaches — kết hợp adaptive và predictive khi các deliverable khác nhau cần cách làm việc khác nhau trên cùng một dự án.

The Standard for PM, p.65 Development Approach Câu 16 — Practice

1. Hybrid Approach (PMBOK 8)

Hybrid development approach combines adaptive and predictive, incorporating elements from both — ví dụ hai deliverable chính, một adaptive một predictive.

Định nghĩa chính thức (PMBOK p.65)

A hybrid development approach combines adaptive and predictive approaches, incorporating elements from both. For example, a project may have two main deliverables, where one is developed using an adaptive approach and the other using a predictive approach.

Software (Adaptive) Constantly evolving requirements · iterations · feedback
+
Data center (Predictive) Upfront planning · formal change control
Brought together in a final Sprint at the end → Hybrid

Keywords trong đề thi → Hybrid

  • Both evolving requirements AND upfront planning / change control
  • Two main deliverables / components with different natures
  • Software agile + infrastructure waterfall
  • Integration sprint / final sprint to merge streams

3. Câu hỏi thực hành — Question 16

Practice Questions PMBOK 8th Ed · Q16
Ethan is working on a project at West Gate Group that will deliver a new enterprise platform. The software component has constantly evolving requirements, while the data center infrastructure is built using upfront planning and a formal change control approach, with the two brought together in a final Sprint at the end. Which development approach should Ethan use?

4. Phân tích đáp án — Đáp án đúng: B

Tại sao chọn B?

  • Software = adaptive signals (evolving requirements).
  • Data center = predictive signals (upfront planning, formal change control).
  • Cùng một project, hai deliverables — textbook hybrid (p.65).
  • Final Sprint integrates both streams — vẫn là hybrid, không chỉ adaptive.

Loại trừ từng đáp án

Đáp ánTại sao không phải
A. Predictive Bỏ qua phần software evolving — chỉ mô tả một nửa scenario.
C. Incremental Deliver từng phần — không capture adaptive + predictive combo.
D. Adaptive Bỏ qua data center upfront planning + formal CC — chỉ một nửa.

Rule of thumb

Một approach keyword → Q14 hoặc Q15 · Hai approach keywords trong cùng project → Hybrid

5. Drill — Approach nào?

Software agile + infrastructure waterfall on same project
Phase gates only, all scope defined early
Short iterations and Product Owner only
One deliverable adaptive, another predictive

0 / 4 đúng

“Single” = chỉ predictive hoặc chỉ adaptive

6. Bẫy thi

Bẫy 1: Chọn D — vì có “Sprint”

Final Sprint là integration point — đề cũng nhấn upfront planning + formal change control cho infra → cần Hybrid.

Bẫy 2: Chọn A — vì enterprise platform “lớn”

Quy mô không quyết định approach — đọc how each component is developed.

Ôn nhanh Q14–16

Predictive · Adaptive · Hybrid — ba câu liên tiếp trong series, pattern rất hay gặp trên đề PMP.

7. Flashcard

Nhấn để lật

Hybrid approach = gì? (p.65)
Combines adaptive + predictive — e.g. two deliverables, one of each approach.
Q16 đáp án?
B — Hybrid (software evolving + data center predictive).

8. Cheat sheet
